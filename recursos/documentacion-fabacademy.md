# Guía de documentación tipo FabAcademy

## Estructura recomendada por práctica

```text
/practica-nombre
  index.md
  img/
  video/
  codigo/
  datos/
```

## Secciones mínimas

1. Objetivo.
2. Diseño de arquitectura.
3. Materiales y software.
4. Procedimiento.
5. Código.
6. Resultados.
7. Errores y depuración.
8. Conclusión.
9. Referencias.

## Evidencia de calidad

Una buena práctica no sólo muestra el resultado final. Muestra el proceso: capturas de error, pruebas fallidas, decisiones de diseño, mediciones y justificación técnica.

## Formato de código

Usar bloques con lenguaje:

```python
print("hola")
```

## Formato para videos

Subir videos a `assets/videos/` o a una plataforma externa y embeberlos con HTML.
