# Documentación tipo FabAcademy

El curso usa una bitácora técnica continua. Cada estudiante o equipo debe mantener un repositorio que permita reconstruir el proceso, no solo mostrar el resultado final.

## Estructura mínima por práctica

```text
/practicas
  /p01-tendencias
  /p02-benchmark-ollama
  /p03-chatbot-local
  /p04-prompting-json
  /p05-mqtt-robot
  /p06-api-comercial
  /p07-vision
  /p08-agente
/proyecto
  /avance-01
  /avance-02
  /demo-final
```

## Checklist del reporte de práctica

- [ ] **Título y fecha.**
- [ ] **Objetivo.** Qué se buscó construir, medir o comprobar.
- [ ] **Contexto.** Por qué esta práctica importa para IA, automatización o robótica.
- [ ] **Materiales y software.** Equipo, sistema operativo, versiones, librerías, modelo usado.
- [ ] **Arquitectura.** Diagrama del flujo de datos o bloques del sistema.
- [ ] **Procedimiento.** Pasos reproducibles.
- [ ] **Código.** Fragmentos relevantes y enlace al archivo completo.
- [ ] **Evidencia.** Capturas, video, tabla, log, gráfica o demo.
- [ ] **Resultados.** Métricas y observaciones.
- [ ] **Errores y depuración.** Qué falló y cómo se resolvió.
- [ ] **Análisis.** Qué significan los resultados.
- [ ] **Implicación prospectiva.** Qué oportunidad, riesgo o tendencia se observa.
- [ ] **Conclusión.** Qué se aprendió y qué se haría diferente.
- [ ] **Referencias.** Fuentes formales o documentación oficial.

## Reglas de evidencia

Una práctica sin evidencia no se considera reproducible. La evidencia puede ser:

- Captura de terminal.
- Captura de interfaz.
- Tabla CSV.
- Gráfica.
- Video de 30 a 90 segundos.
- Log de ejecución.
- Diagrama de arquitectura.
- Fragmento de código con explicación.
