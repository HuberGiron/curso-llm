# Sesión 11: Ollama como API local, MQTT y WebSocket

**Semana:** 3  
**Fecha:** 8 de Junio  
**Tema GEM:** Aplicación tecnológica  
**Subtema GEM:** Conceptos / Herramientas / Casos

## Objetivo de la sesión

Desarrollar los elementos conceptuales y prácticos relacionados con **Ollama como API local, MQTT y WebSocket**, conectándolos con el proyecto integrador de IA con LLM para automatización y robótica.

## Contenido del profesor

Explicación de integración entre LLM local, backend, frontend y sistemas de control; REST, MQTT, WebSocket, tópicos, QoS, latencia y trazabilidad.

Demostración del flujo LLM → JSON → MQTT/WebSocket → actuador físico o virtual; criterios de prueba para comunicación y control.

## Actividad del alumnado

Diseñar el flujo de comunicación del proyecto y proponer tabla de tópicos MQTT, endpoints o mensajes WebSocket.
