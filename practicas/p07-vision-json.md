# Visión a salida estructurada

<div class="callout"><strong>Objetivo:</strong> Convertir imagen, sensor o estado visual en decisión estructurada.</div>

**Sesiones relacionadas:** 19

## Instrucciones

1. Crear una página nueva en el repositorio de práctica.
2. Documentar el objetivo y la arquitectura.
3. Ejecutar la actividad técnica indicada por el profesor.
4. Guardar evidencia de ejecución.
5. Analizar resultados y fallas.
6. Cerrar con reflexión prospectiva.

## Checklist del reporte

- [ ] Título, fecha e integrantes.
- [ ] Objetivo de la práctica.
- [ ] Diagrama de arquitectura o flujo.
- [ ] Materiales, software, modelo y versiones.
- [ ] Procedimiento paso a paso.
- [ ] Código o enlace al código.
- [ ] Evidencia de ejecución: capturas, logs, CSV, gráfica o video.

- [ ] Referencias.

## Criterios de revisión

La práctica se evaluará por reproducibilidad, claridad técnica, calidad de evidencia, análisis y vínculo con automatización/robótica.
